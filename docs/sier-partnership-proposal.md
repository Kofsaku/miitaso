## 御社が抱えている（かもしれない）課題

| 課題 | よくある状況 |
|---|---|
| 提案段階でデザインの見せ方が弱い | 機能要件は詳しいが、画面イメージが伝わらず受注を逃す |
| 既存システムの改修案件で「見た目が変わらない」と言われる | ロジックは改善しても、UIが古いままでクライアントの満足度が上がらない |
| デザイナーを正社員で抱えるほどの案件量がない | 案件ごとに外注するが、毎回探すのが手間 |
| クライアントから「今風にしてほしい」と言われるが対応しきれない | モバイル対応・UX改善の知見が社内にない |

## ご提案内容：デザインパートナー契約

### 基本構造

```
御社のクライアント → 御社（開発・PM） → miitaso（UI/UXデザイン）
```

- 御社がクライアントとの窓口・開発を担当
- miitasoはUI/UXデザインに特化して御社を支援
- クライアントから見ると「御社のデザイン力が上がった」ように見える

### miitasoが担当する範囲

| フェーズ | 担当内容 |
|---|---|
| 提案段階 | 画面モックアップ作成（Figma）、Before/Afterビジュアル |
| 要件定義 | 画面遷移図、ワイヤーフレーム作成 |
| デザイン | UIデザイン（レスポンシブ対応）、デザインシステム構築 |
| 実装支援 | フロントエンド実装（HTML/CSS/React等）、コーディングガイド提供 |
| 納品後 | デザインの微調整・改善提案（オプション） |

### miitasoが担当しない範囲

- サーバーサイド開発（御社の領域）
- インフラ構築・運用
- クライアントとの直接交渉（御社経由）

| 規模 | デザイン費用目安 | 含まれる範囲 |
|---|---|---|
| 小規模（5画面以下） | 30〜50万円 | ワイヤーフレーム + UIデザイン + フロント実装 |

| 大規模（20画面以上） | 150万円〜 | 同上 + 段階的デザイン改善 |

### B案：月額パートナー契約（推奨）

| プラン | 月額 | 稼働目安 | 内容 |
|---|---|---|---|
| ライト | 15万円 | 月20時間 | 小規模案件1件 or 提案用モック複数件 |
| スタンダード | 30万円 | 月40時間 | 中規模案件1件を継続対応 |
| プレミアム | 50万円 | 月70時間 | 複数案件を並行対応 |

**月額契約のメリット（御社にとって）：**
- 案件ごとにデザイナーを探す手間がなくなる
- 提案段階から気軽にモックアップを依頼できる（＝受注率向上）
- 御社のクライアントへの提案力が恒常的に上がる
